# Contributing to Pong Game

Thank you for your interest in contributing! Here are some guidelines to help you get started.

## 🤝 How to Contribute

### Reporting Bugs

If you find a bug, please create an issue with:
- A clear, descriptive title
- A detailed description of the bug
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Any relevant screenshots or error messages

### Suggesting Features

Have an idea for a new feature? Please open an issue with:
- A clear description of the feature
- Why you think it would be useful
- Any relevant examples or mockups

### Submitting Changes

1. **Fork the repository** - Click the "Fork" button on GitHub
2. **Clone your fork** - `git clone https://github.com/your-username/pongGame.git`
3. **Create a branch** - `git checkout -b feature/your-feature-name`
4. **Make your changes** - Edit the files as needed
5. **Test your changes** - Ensure the game still works correctly
6. **Commit your changes** - `git commit -m "Add feature: brief description"`
7. **Push to your fork** - `git push origin feature/your-feature-name`
8. **Create a Pull Request** - Submit your PR on GitHub with a clear description

## 📋 Code Guidelines

### HTML/CSS/JavaScript Standards

- Use clear, descriptive variable and function names
- Add comments for complex logic
- Keep functions focused and modular
- Maintain consistent indentation (2 or 4 spaces)
- Test your code in multiple browsers

### Game Logic Changes

When modifying game mechanics:
- Document what you changed and why
- Ensure the game remains balanced and playable
- Test AI difficulty thoroughly
- Verify collision detection works correctly

## 🎨 Design Guidelines

- Maintain the pastel yellow and cream color scheme
- Ensure the UI remains responsive
- Test changes on mobile and desktop
- Keep the overall aesthetic cohesive

## 📝 Commit Message Guidelines

Use clear, descriptive commit messages:
- ✅ Good: `Add difficulty settings and pause feature`
- ❌ Bad: `fix stuff` or `update`

Format: `[Type] Brief description`

Types:
- `[Feature]` - New feature
- `[Fix]` - Bug fix
- `[Improvement]` - Code improvement
- `[Docs]` - Documentation
- `[Style]` - CSS/styling changes

## 🧪 Testing

Before submitting a PR, please test:
- [ ] Game starts and stops correctly
- [ ] Ball physics work as expected
- [ ] Collision detection is accurate
- [ ] AI behaves appropriately
- [ ] UI is responsive on mobile
- [ ] No console errors

## ✨ Ideas for Contributions

Looking for something to work on? Consider:

- **Difficulty Settings** - Add Easy/Medium/Hard modes
- **Sound Effects** - Add audio feedback for collisions and scoring
- **Visual Effects** - Particle effects, animations
- **Multiplayer** - Two-player mode
- **Power-ups** - Special items that affect gameplay
- **Themes** - Additional color schemes
- **Documentation** - Improve comments in code
- **Testing** - Add unit tests

We appreciate all contributions, big and small. Your help makes this project better!
